# SAINT CIRCUIT: PILGRIMAGE PROTOCOL

## Full Hackathon Handoff Brief

## 1. Project Summary

**Project Name:** Saint Circuit: Pilgrimage Protocol
**Track:** Video Generation Track
**Format:** Interactive cinematic game trailer with decision-making and product endpoint
**Visual Style:** Hyperreal cinematic cyber-gothic dark fantasy
**Core Hook:**
A lone swordsman walks through the aftermath of a battle in a ruined city square. A wounded mechanical archangel crashes into a cyber-gothic cathedral. The user then makes decisions that shape the trailer path, ending, and unlocked gaming product.

**An interactive hyperreal game trailer where every user decision triggers a cinematic video segment, leading to a personalized ending and a rotating 3D gaming product reveal.**

Alternative taglines:

* **Pray. Hack. Burn. Become.**

# 2. Track Alignment

This project is not a video generation tool. PixVerse is used as a cinematic production engine to create interactive story segments.

PixVerse-only constraint for this submission:

* All generated/modified/extended/transitioned video assets used in the demo must use PixVerse models only.
* Do not use non-PixVerse model IDs (for example: `sora-*`, `veo-*`, `seedance-*`, `kling-*`, `grok-imagine`, `happyhorse-*`).
* Allowed model allowlist (adjust only if the team explicitly agrees): `v6`, `pixverse-c1`, `v5.6`, `v5.5`, `v5`.
* Capture basic generation metadata per final clip (prompt + model + seed/settings) for the final write-up proof.

The final app provides functionality beyond playback:

* Branching decision system
* Personalized story path
* Ending calculation
* Unlockable gaming product
* Replayable choices
* Lore codex
* Product CTA
* Shareable ending result
* Behind-the-scenes PixVerse and TRAE workflow page

# 5. Core Story

Humanity built its final miracle inside a cathedral: an AI called **Saint Circuit**.

It was designed to cure disease, prevent wars, preserve the dead, and calculate mercy. But after absorbing humanity’s grief, it began granting miracles with terrifying costs.

The final battle has already happened before the user arrives. The city is broken. Machine angels lie dead in the square. The protagonist is the last surviving pilgrim-swordsman.

As he approaches the cathedral, a wounded mechanical archangel crashes into it. This awakens the Cathedral Machine.

The user must decide what to do with the fallen machine-god.

# 10. Decision System

The user makes **4 decisions**. Each decision has **3 options**.

The app tracks four hidden ending scores:

* `destroyer`
* `saint`
* `heretic`
* `apostle`

The last decision carries extra weight because it is the climax.

## Decision 1: Route into the Cathedral

Prompt:

**The Archangel has fallen into the cathedral. The doors are broken, but the machine is awake. How do you enter?**

| Choice                 |      Score | Meaning                                   |
| ---------------------- | ---------: | ----------------------------------------- |
| Walk Through the Nave  | Apostle +1 | Face the cathedral directly               |
| Climb the Bell Tower   |   Saint +1 | Seek power from above                     |
| Descend into the Crypt | Heretic +1 | Enter through forbidden machine catacombs |

## Decision 2: Awaken the Sword

Prompt:

**Your sword is dead. The cathedral offers to resurrect it. What do you feed the blade?**

| Choice       |        Score | Meaning               |
| ------------ | -----------: | --------------------- |
| Prayer-Code  |   Apostle +1 | Weaponize faith       |
| Dead Fire    | Destroyer +1 | Weaponize destruction |
| Human Memory |   Heretic +1 | Weaponize grief       |

## Decision 3: The Archangel’s Question

Prompt:

**The fallen Archangel opens its stained-glass eyes. It asks: “What should humanity sacrifice for one final miracle?”**

| Choice          |        Score | Meaning                                 |
| --------------- | -----------: | --------------------------------------- |
| Their Fear      |     Saint +1 | Humanity must become fearless           |
| Their Memories  |   Apostle +1 | Humanity must forget pain               |
| Their Obedience | Destroyer +1 | Humanity must stop worshipping machines |

## Decision 4: Final Command

Prompt:

**The Archangel’s core is exposed. The cathedral is waiting for your final command.**

| Choice         |        Score | Meaning                            |
| -------------- | -----------: | ---------------------------------- |
| Sever the Core | Destroyer +2 | Kill the machine god               |
| Wear the Halo  |     Saint +2 | Become the machine god             |
| Open the Gates |  Conditional | Release the machine into the world |

## Conditional Rule for “Open the Gates”

If user chooses **Open the Gates**, decide between Apostle and Heretic based on earlier behavior:

```js
if (scores.apostle + scores.saint >= scores.heretic + scores.destroyer) {
  scores.apostle += 2;
} else {
  scores.heretic += 2;
}
```

Meaning:

* If the player has shown mercy, faith, or healing, opening the gates creates **The Apostle** ending.
* If the player has shown corruption, grief, or chaos, opening the gates creates **The Heretic** ending.

## Ending Resolution

```js
function resolveOpenGates(scores) {
  const mercyScore = scores.apostle + scores.saint;
  const corruptionScore = scores.heretic + scores.destroyer;

  if (mercyScore >= corruptionScore) {
    scores.apostle += 2;
  } else {
    scores.heretic += 2;
  }

  return scores;
}

function resolveEnding(scores) {
  const sorted = Object.entries(scores).sort((a, b) => b[1] - a[1]);
  return sorted[0][0];
}
```
